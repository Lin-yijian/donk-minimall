# MiniMall 项目文档

> 全栈微型电商系统 | Next.js 16 + Prisma 5 + SQLite + Tailwind CSS 4

## 一、项目概况

MiniMall 是一套完整的微型电商系统，涵盖前台商城（商品浏览、购物车、订单）和后台管理（商品/分类/订单 CRUD），并实现了**星悦会员等级体系**（按累计消费自动升级，享受阶梯折扣）。

| 指标 | 数值 |
|------|------|
| 源码文件 | 42 个 TS/TSX |
| 测试文件 | 5 个，52 个测试用例 |
| API 路由 | 10 个（cart、orders、admin/products、admin/categories、admin/orders） |
| 页面路由 | 10 个（首页、商品列表/详情、登录/注册、购物车、订单列表/详情、后台 4 页） |
| 数据库模型 | 6 个（User、Category、Product、Cart、Order、OrderItem） |
| Prisma 版本 | 5.22.0 |
| 测试框架 | Vitest 2.1 |

## 五、功能模块

### 5.1 商品浏览（公开）

| 页面 | 路由 | 功能 |
|------|------|------|
| 首页 | `/` | Banner + 4 个分类导航 + 6 个精选商品 |
| 商品列表 | `/products` | 搜索框 + 分类筛选 + 12 个商品网格 + 分页 |
| 商品详情 | `/products/[slug]` | 大图 + 价格 + 库存 + 加入购物车 + 商品描述 |

### 5.2 用户认证（公开）

| 页面 | 路由 | 功能 |
|------|------|------|
| 注册 | `/auth/register` | 用户名 + 邮箱 + 密码 → Zod 校验 → bcrypt → Prisma 写入 |
| 登录 | `/auth/login` | 邮箱 + 密码 → Auth.js Credentials → JWT Cookie |

**认证流程：**
```
注册 → redirect 登录页(registered=true) → 登录 → signIn("credentials") 
→ authorize 查库 + bcrypt 验证 → JWT token → cookie → redirect /
```

### 5.3 购物车（需登录）

| 操作 | API | 说明 |
|------|-----|------|
| 查看 | `GET /api/cart` | 获取当前用户购物车列表 |
| 加入 | `POST /api/cart` | 同商品自动叠加数量，更新价格快照 |
| 修改 | `PUT /api/cart/[id]` | 修改购买数量 |
| 删除 | `DELETE /api/cart/[id]` | 移除购物车项 |

**购物车页面** (`/cart`)：商品图片 + 名称 + 单价 + 数量 ± 按钮 + 小计 + 删除 + 底部总价 + 提交订单按钮

### 5.4 订单（需登录）

| 操作 | API | 说明 |
|------|-----|------|
| 下单 | `POST /api/orders` | 读取购物车 → 计算金额(含折扣) → 模拟支付 → 创建订单 → 扣库存 → 清空购物车 → 更新累计消费 → 重算会员等级 |
| 列表 | `GET /api/orders` | 当前用户所有订单（倒序） |
| 详情 | `GET /api/orders/[id]` | 订单明细 + 金额明细 |

**下单流程：**
```
购物车 → POST /api/orders → 
  1. 验证登录 + 购物车非空 + 库存充足
  2. 计算 subtotal（商品小计）
  3. 读取 membershipTier → 查折扣率 → 计算 discount → total
  4. 模拟支付（生成 MOCK_xxx 交易号）
  5. 创建 Order + OrderItems（快照商品信息）
  6. 扣减 stock
  7. 清空 cart
  8. 更新 totalSpent → 重算 membershipTier
  9. 返回 orderId → 跳转订单详情
```

### 5.5 星悦会员体系

| 等级 | 累计消费 | 折扣 |
|------|---------|------|
| 普通会员 | < ¥800 | 原价 |
| ⭐ 星悦一级 | ≥ ¥800 | 9.8 折 |
| ⭐⭐ 星悦二级 | ≥ ¥8,000 | 9 折 |
| ⭐⭐⭐ 星悦三级 | ≥ ¥80,000 | 8.8 折 |

**核心规则：**
- **升级依据**：历史已支付订单的 `total` 累加到 `User.totalSpent`
- **等级判定**：`calculateTier(totalSpent)` — 根据累计消费实时计算
- **只升不降**：等级不会下降
- **折扣生效时机**：仅有后续订单享受新等级折扣（触发升级的当前订单按当前等级折扣）
- **实现文件**：`src/lib/membership.ts`

### 5.6 后台管理（管理员）

| 页面 | 路由 | 功能 |
|------|------|------|
| 仪表盘 | `/admin` | 商品总数、订单总数、用户总数、总营收 4 个统计卡片 |
| 商品管理 | `/admin/products` | 表格列表 + 新增/编辑弹窗 + 删除 |
| 订单管理 | `/admin/orders` | 订单列表 + 状态下拉变更（PENDING→SHIPPED→DELIVERED→CANCELLED） |
| 分类管理 | `/admin/categories` | 表格列表 + 新增/编辑弹窗 + 删除 |

**权限控制：**
- `middleware.ts` → 检查 JWT → 非 ADMIN → `redirect("/")`
- `AdminLayout` → RSC 验证 → `session.user.role !== "ADMIN"` → 拒绝
- 所有 Admin API → 检查 `auth().user.role !== "ADMIN"` → 403

## 六、权限体系

| 路由 | 权限 | 守卫方式 |
|------|------|---------|
| `/`, `/products`, `/products/[slug]` | 公开 | 无 |
| `/auth/login`, `/auth/register` | 公开（已登录用户自动跳转首页） | 无 |
| `/cart`, `/orders`, `/orders/[id]` | 需登录 | Middleware → `/auth/login?callbackUrl=...` |
| `/admin/*` | 需 ADMIN 角色 | Middleware → `/` + Layout RSC 验证 |
| `/api/cart/*`, `/api/orders/*` | 需登录 | API 内 `auth()` 检查 → 401 |
| `/api/admin/*` | 需 ADMIN 角色 | API 内 `auth().user.role` 检查 → 403 |

## 九、架构决策

| 决策 | 理由 |
|------|------|
| SQLite（非 PostgreSQL） | 微型项目零配置，单文件部署 |
| JWT Session（非 Database Session） | 减少数据库查询，性能更好 |
| 购物车冗余快照 | 商品价格变更后购物车内容保持一致 |
| 订单明细冗余快照 | 历史订单不受商品修改影响 |
| Server Components 默认 | 减少客户端 JS，直接读库 |
| API Routes（非纯 Server Actions） | 购物车/订单的交互式 UI 需要 REST 端点 |
| Int 主键 | SQLite 下简单高效，用 parseInt 解析 |
| 模拟支付 | 免去真实支付集成复杂度，专注电商核心逻辑 |
| Zod v4 | 使用 `issues` 替代 v3 的 `errors` |
